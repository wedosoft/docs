# 티켓, 문제, 변경, 릴리스에서 종속 필드 사용하기

종속 필드(Dependent Fields)를 사용하면 한 필드의 선택에 따라 다른 필드의 옵션이 동적으로 변경되어 더 정확하고 체계적인 정보 수집이 가능합니다.

## 종속 필드란?

종속 필드는 상위 필드의 값에 따라 하위 필드의 옵션이 자동으로 필터링되거나 변경되는 기능입니다. 이를 통해 계층적이고 논리적인 정보 분류가 가능합니다.

### 종속 필드의 장점

- **정확한 분류**: 체계적인 카테고리 분류로 정확한 정보 수집
- **사용자 편의성**: 관련 있는 옵션만 표시하여 선택 용이성 증대
- **데이터 품질**: 논리적으로 일관된 데이터 입력 보장
- **자동화 지원**: 분류에 따른 자동 할당 및 라우팅 가능

## 종속 필드 설정

### 기본 설정 절차

1. **관리자 → 양식 필드**로 이동합니다.
2. 대상 양식(티켓/문제/변경/릴리스)을 선택합니다.
3. **종속 필드 설정**을 클릭합니다.
4. 상위 필드와 하위 필드를 선택합니다.

### 상위 필드 설정

상위 필드는 하위 필드의 옵션을 결정하는 기준이 되는 필드입니다:

#### 적합한 상위 필드
- **카테고리**: 주요 분류 기준
- **부서**: 조직 구조 기반 분류
- **제품/서비스**: 제공 서비스 기반 분류
- **우선순위**: 중요도 기반 분류

#### 상위 필드 옵션 설정
```
예시: IT 카테고리 상위 필드
- 하드웨어
- 소프트웨어  
- 네트워크
- 보안
- 기타
```

### 하위 필드 매핑

각 상위 필드 옵션에 따른 하위 필드 옵션을 설정합니다:

#### 하드웨어 하위 카테고리
```
상위: 하드웨어
하위 옵션:
- 데스크톱 컴퓨터
- 노트북
- 프린터
- 모니터
- 키보드/마우스
```

#### 소프트웨어 하위 카테고리
```
상위: 소프트웨어
하위 옵션:
- 운영체제
- 오피스 프로그램
- 개발 도구
- 보안 소프트웨어
- 기타 응용 프로그램
```

## 실제 활용 사례

### IT 헬프데스크 분류

#### 3단계 분류 체계
```
1단계: 주요 카테고리
- IT 인프라
- 애플리케이션
- 사용자 지원
- 보안

2단계: 세부 카테고리
IT 인프라 → 서버, 네트워크, 스토리지
애플리케이션 → ERP, CRM, 이메일, 협업도구

3단계: 구체적 항목
서버 → 웹서버, DB서버, 파일서버
네트워크 → 스위치, 라우터, 방화벽
```

### 인사 서비스 분류

#### 인사 업무별 분류
```
1단계: 인사 영역
- 급여/복리후생
- 교육/개발
- 채용/인사이동
- 노무/법무

2단계: 세부 업무
급여/복리후생 → 급여계산, 휴가관리, 보험, 퇴직금
교육/개발 → 신규교육, 역량개발, 외부교육, 평가
```

### 시설 관리 분류

#### 시설별 요청 분류
```
1단계: 시설 유형
- 사무 공간
- 회의실
- 공용 시설
- 보안 시설

2단계: 요청 유형
사무 공간 → 좌석배정, 이사, 청소, 수리
회의실 → 예약, 장비, 설정, 문제신고
```

## 고급 종속 필드 설정

### 다중 레벨 종속성

3단계 이상의 복잡한 종속 관계 설정:

```
1단계: 제품군 (ERP)
2단계: 모듈 (재무, 인사, 생산)
3단계: 기능 (예산관리, 급여계산, 생산계획)
4단계: 세부 기능 (월별예산, 연말정산, 작업지시)
```

### 조건부 종속성

특정 조건에서만 종속 관계가 활성화되도록 설정:

```
조건: 우선순위가 "긴급"인 경우
종속성: 긴급 대응팀 목록만 표시
일반: 모든 팀 목록 표시
```

### 크로스 필드 종속성

여러 필드의 값에 따라 옵션이 결정되는 복합 종속성:

```
조건: 부서 = "IT" AND 위치 = "서울"
결과: 서울 IT팀 담당자 목록만 표시
```

## 자동화 연동

### 할당 규칙 자동화

종속 필드 값에 따른 자동 할당:

```
규칙: 카테고리가 "네트워크"이고 하위카테고리가 "방화벽"
작업: 네트워크 보안팀에 자동 할당
```

### SLA 정책 적용

분류에 따른 자동 SLA 적용:

```
규칙: 카테고리가 "서버"이고 우선순위가 "높음"
작업: 2시간 응답 SLA 적용
```

### 알림 규칙 설정

특정 분류에 대한 특별 알림:

```
규칙: 카테고리가 "보안"
작업: 보안팀장에게 즉시 알림 발송
```

## 데이터 분석 및 보고서

### 분류별 통계

종속 필드를 활용한 상세 분석:

#### 카테고리별 요청 분포
- 1단계 카테고리별 티켓 수
- 2단계 세부 분류별 해결 시간
- 3단계 구체적 항목별 만족도

#### 트렌드 분석
- 월별 카테고리 변화
- 계절별 요청 패턴
- 부서별 요청 특성

### 보고서 생성

종속 필드 데이터를 활용한 보고서:

```
보고서 예시: IT 서비스 분석
- 하드웨어 vs 소프트웨어 요청 비율
- 긴급 요청의 카테고리별 분포
- 해결 시간의 분류별 차이
```

## 모범 사례

### 분류 체계 설계

#### 계층 구조 원칙
- **상호 배타적**: 각 옵션이 겹치지 않도록 설계
- **완전 포괄적**: 모든 경우를 포함하는 분류 체계
- **직관적**: 사용자가 쉽게 이해할 수 있는 구조
- **확장 가능**: 향후 추가 분류가 용이한 구조

#### 명명 규칙
- **일관된 용어**: 동일한 의미는 같은 용어 사용
- **명확한 표현**: 모호하지 않은 구체적 표현
- **표준 용어**: 업계 표준이나 사내 표준 용어 사용

### 사용자 교육

#### 상담원 교육
- 분류 체계의 이해와 활용법
- 정확한 분류의 중요성 인식
- 예외 상황 처리 방법

#### 요청자 안내
- 셀프 서비스에서의 올바른 분류 선택
- 분류가 서비스 품질에 미치는 영향
- 불확실한 경우 상담원 문의 안내

### 지속적인 개선

#### 정기적 검토
- **월간**: 분류 사용 현황 분석
- **분기별**: 분류 체계 적절성 검토
- **연간**: 전체 분류 체계 재설계 검토

#### 피드백 수집
- 상담원의 분류 체계 개선 의견
- 요청자의 분류 선택 어려움 사항
- 관리자의 분석 및 보고서 요구사항

:::tip 효과적인 종속 필드 활용
종속 필드는 단순한 분류를 넘어 자동화, 분석, 보고서의 기반이 됩니다. 처음에는 간단한 2단계 구조로 시작하여 점진적으로 확장하는 것이 좋습니다.
:::

:::warning 주의사항
너무 복잡한 종속 관계는 사용자에게 혼란을 줄 수 있습니다. 실제 업무 흐름을 반영하면서도 직관적으로 이해할 수 있는 수준으로 설계하세요.
:::
