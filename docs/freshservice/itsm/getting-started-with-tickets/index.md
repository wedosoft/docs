# 티켓 시작하기 종합 가이드

Freshservice의 티켓 시스템은 IT 서비스 요청부터 인시던트 해결까지 모든 과정을 체계적으로 관리하는 핵심 도구입니다. 신규 사용자도 쉽게 시작할 수 있도록 단계별로 안내합니다.

:::info 티켓 시스템 시작 전 준비사항
- 계정 접근 권한 확인: 상담원 또는 요청자 권한 필요
- 기본 설정 완료: 프로필 정보, 알림 설정, 접근 가능한 서비스 카탈로그 확인
- 회사 정책 숙지: 티켓 유형별 처리 절차 및 우선순위 기준 이해
:::

## 티켓 유형별 생성 가이드

### 인시던트 티켓 (Incident)

서비스 중단이나 장애 상황에서 사용하는 티켓 유형입니다.

1. **대시보드에서 'New Ticket' 선택**
2. **티켓 유형을 'Incident'로 설정**
3. **긴급도에 따른 우선순위 선택**:
   - Critical: 전체 서비스 중단
   - High: 핵심 기능 영향
   - Medium: 부분적 기능 제한
   - Low: 일반적 문의

:::warning 인시던트 우선순위 주의사항
Critical 등급은 실제 비상상황에만 사용하세요. 잘못된 분류는 대응 체계에 혼란을 야기할 수 있습니다.
:::

### 서비스 요청 (Service Request)

일반적인 IT 서비스 요청이나 승인이 필요한 업무에 사용합니다.

1. **서비스 카탈로그에서 해당 서비스 선택**
2. **필수 정보 입력**: 요청 사유, 기대 일정, 추가 요구사항
3. **승인자 지정** (필요시)
4. **관련 자산 연결** (해당시)

### 문제 (Problem)

반복적으로 발생하는 인시던트의 근본 원인을 해결하기 위한 티켓입니다.

:::tip 문제 티켓 활용법
동일한 유형의 인시던트가 3회 이상 발생하면 문제 티켓으로 근본 원인 분석을 진행하세요.
:::

## 티켓 생성부터 완료까지 프로세스

### 1단계: 티켓 생성

**웹 포털 사용법:**
1. Freshservice 포털 로그인
2. 'Submit a Request' 또는 'New Ticket' 클릭
3. 요청 유형 선택 및 세부 정보 입력

**이메일 사용법:**
- 지정된 이메일 주소로 요청 내용 전송
- 제목은 자동으로 티켓 제목이 됨
- 첨부파일 자동 포함

### 2단계: 티켓 할당 및 처리

1. **자동 할당**: 설정된 규칙에 따라 담당자 배정
2. **상태 업데이트**: Open → In Progress → Pending → Resolved
3. **커뮤니케이션**: 담당자와 요청자 간 실시간 소통

### 3단계: 해결 및 피드백

1. **해결 확인**: 담당자가 문제 해결 후 상태 변경
2. **사용자 검증**: 요청자의 해결 여부 확인
3. **만족도 조사**: 서비스 품질 개선을 위한 피드백

:::success 프로세스 완료
모든 단계를 거쳐 티켓이 성공적으로 처리되었습니다. 동일한 문제 발생 시 기존 티켓을 참조하여 빠른 해결이 가능합니다.
:::

## 실무 활용 예시

### 상황 1: 대기업 IT 헬프데스크 운영
**목표**: 일일 100건 이상의 다양한 IT 요청을 효율적으로 처리
**방법**:
1. 카테고리별 자동 분류 시스템 구축
2. 우선순위 기반 에스컬레이션 설정
3. 셀프 서비스 포털을 통한 일반 요청 자동화
4. 대시보드를 활용한 실시간 모니터링

**결과**: 평균 해결 시간 40% 단축, 사용자 만족도 90% 이상 유지

### 상황 2: 중소기업 통합 서비스 데스크
**목표**: 소수 인원으로 IT와 일반 업무 지원 통합 관리
**방법**:
1. 간소화된 티켓 양식으로 접근성 향상
2. FAQ와 연동된 자동 응답 시스템
3. 모바일 앱을 활용한 현장 즉시 대응
4. 월간 리포트로 서비스 개선점 도출

**결과**: 1인당 처리 효율성 3배 증가, 즉시 해결율 70% 달성

## 성공적인 티켓 활용을 위한 팁

### 효과적인 티켓 작성법

1. **명확한 제목**: 문제나 요청 내용을 한 눈에 파악 가능
2. **구체적인 설명**: 언제, 어디서, 무엇을, 어떻게 발생했는지 명시
3. **스크린샷 첨부**: 오류 메시지나 화면 상태 이미지 포함
4. **우선순위 설정**: 업무 영향도에 따른 적절한 분류

### 빠른 해결을 위한 정보 준비

- **에러 코드나 메시지**: 정확한 오류 내용
- **재현 단계**: 문제가 발생하는 구체적인 과정
- **시스템 환경**: 운영체제, 브라우저, 소프트웨어 버전
- **관련자 정보**: 영향받는 사용자나 부서

:::tip 프로 팁
자주 사용하는 요청은 즐겨찾기에 추가하고, 템플릿을 활용하면 훨씬 빠르게 티켓을 생성할 수 있습니다.
:::

## 문제 해결

### 자주 발생하는 문제

#### 문제: 티켓을 찾을 수 없음
**원인**: 검색 조건 설정 오류 또는 권한 부족
**해결**:
1. 티켓 번호나 제목으로 정확한 검색
2. 날짜 범위 확장하여 재검색
3. 관리자에게 접근 권한 문의

#### 문제: 알림을 받지 못함
**원인**: 이메일 설정 또는 스팸 필터 이슈
**해결**:
1. 프로필의 알림 설정 확인
2. 이메일 주소 정확성 검증
3. 스팸함 및 필터 규칙 점검
